Action Management Prototype

How the admin experience adapts based on whether the org has existing Service Cloud traffic (warm-start) or is configuring Agentforce for the first time (cold-start).

🔥

Warm-Start

Existing Service Cloud Customer

Org has 30+ days of case/chat data. Traffic patterns, resolution history, and query baselines exist.

  • Traffic-driven recommendations
  • Real metrics with regression detection
  • Bulk import test cases from logs
  • Historical baseline comparison
❄️

Cold-Start

Net New Customer

No prior Service Cloud traffic. New to Salesforce or migrating from another platform.

  • Industry starter packs
  • Guided first publish flow
  • Absolute thresholds (no baseline)
  • Transition nudges as data grows

Key Differences at a Glance

SurfaceWarm-Start (Existing)Cold-Start (Net New)
Recommendations "187 queries/mo hitting fallback — add Order Tracking" "Most Service orgs start with these 5 actions"

Regression Delta Active from first publish (historical baseline) Activates after second publish (v1 → v2 only)
Post-Publish Regression vs. baseline (24h monitoring) Absolute threshold monitoring only
